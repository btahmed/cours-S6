\section{Project Timeline}

The following timeline presents the weekly advancement of the StudentFlow project, from conceptualization to final presentation. Each phase reflects the required steps defined in the course guidelines.

\subsection*{Week 1 – Project Conceptualization}
\begin{itemize}
    \item Define the global goal of the project (student productivity and well-being)
    \item Identify the main problem: irregular schedules, stress, and poor time management among students
    \item Initial brainstorming of the solution (personalized planning platform)
    \item Project description draft
\end{itemize}

\subsection*{Week 2 – Research \& Data Collection}
\begin{itemize}
    \item Research on student stress, sleep deprivation, and academic overload
    \item Collect statistics from academic articles and health organizations
    \item Analyze existing solutions (Apple Watch, Google Calendar, Notion)
    \item Justify why current tools are insufficient for students
\end{itemize}

\subsection*{Week 3 – Target Market \& Marketing Strategy}
\begin{itemize}
    \item Define target audience (university students, irregular schedules)
    \item Define value proposition compared to existing tools
    \item Create project name, slogan, and visual identity
    \item Initial ideas for poster and promotional content
\end{itemize}

\subsection*{Week 4 – Timeline Finalization \& Role Assignment}
\begin{itemize}
    \item Finalize project timeline
    \item Assign roles (design, development, research, presentation)
    \item Define responsibilities within the group
    \item Plan weekly objectives
\end{itemize}

\subsection*{Week 5 – Prototype Design}
\begin{itemize}
    \item Design the application interface (planning grid, dashboard)
    \item Define core features (planning, stress tracking, sleep tracking)
    \item User flow design
    \item Begin low-fidelity prototype
\end{itemize}

\subsection*{Week 6 – Functional Prototype Development}
\begin{itemize}
    \item Develop a functional prototype (web-based)
    \item Implement planning system and personalization logic
    \item Test basic interactions
    \item Internal testing and debugging
\end{itemize}

\subsection*{Week 7 – SWOT Analysis \& Improvements}
\begin{itemize}
    \item Conduct SWOT analysis (Strengths, Weaknesses, Opportunities, Threats)
    \item Identify limitations of the prototype
    \item Improve features based on weaknesses
    \item Prepare evaluation criteria
\end{itemize}

\subsection*{Week 8 – Report Writing (Submission April 2)}
\begin{itemize}
    \item Write final project report
    \item Integrate research, timeline, SWOT, and prototype description
    \item Review and proofread report
    \item Submit report by April 2
\end{itemize}

\subsection*{Week 9 – Presentation Plan \& Promotional Material}
\begin{itemize}
    \item Prepare presentation structure
    \item Design slides and visuals
    \item Create promotional poster and short video concept
    \item Rehearse oral presentation
\end{itemize}

\subsection*{Week 10 – Final Presentation}
\begin{itemize}
    \item Final prototype demonstration
\end{itemize}
